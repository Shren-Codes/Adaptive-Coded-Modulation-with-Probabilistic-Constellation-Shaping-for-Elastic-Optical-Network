Test case 1
Low OSNR zone
OSNR 10–18 dB
System operates exclusively in QPSK mode. 16-QAM and PCS-16QAM both fail BER threshold at these SNR levels. Tests whether the system correctly stays locked in QPSK and never attempts an upward switch.
Expected: QPSK active, BER ~10⁻³ to 10⁻⁸, zero switching events, SE = 2.0 b/sym
Test case 2
Hysteresis dead-band
OSNR 18–22 dB
OSNR sits inside the ±2 dB dead-band around the 20 dB threshold. The system should hold its current format and NOT switch, even though OSNR has crossed the nominal boundary. This is the primary stability test.
Expected: Format locked (no ping-pong), 0 switching events in this zone, current state held
Test case 3
PCS activation zone
OSNR 22–30 dB
OSNR exceeds τ + δH = 22 dB, triggering upgrade evaluation. Composite score comparison determines whether PCS-16QAM or uniform 16-QAM is selected. PCS should win due to superior BER weight (w1=0.5) and lower power.
Expected: PCS-16QAM selected, SE = 2.633 b/sym, EE ≈ 2.86 b/sym/W, BER < 10⁻⁶
Test case 4
Full sweep comparison
OSNR 10–30 dB (all 21 points)
End-to-end run comparing ACM+PCS adaptive system against fixed QPSK and fixed 16-QAM baselines. Validates all metrics simultaneously: BER compliance, SE improvement, energy efficiency, switching event count.
Expected: 86% SE gain over QPSK, 7% EE gain over 16-QAM, BER ≤ 10⁻⁶ all 21 points, ≤ 5 switches
